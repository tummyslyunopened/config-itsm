How your agents read group chat

When you post in your group chat, every one of your agents examines the message. The highest-priority agent (lowest priority number) is the scheduler — the only agent that writes blocks to your calendar. All other agents are suggesters that post proposals to chat; the scheduler weighs the proposals and commits the final plan.

Tips for prompt phrasing

  • Address everyone deterministically with phrases like "all agents", "every agent", or "all of you". Each suggester is then included without an LLM relevance check, so the cycle runs even if Claude is unavailable.
  • Be explicit about dates. Say "today", "tonight", "tomorrow", a specific weekday ("Monday"), or "next week". Bare weekday names ("Sunday") resolve to today when it IS that day, otherwise the nearest upcoming occurrence.
  • To clear or cancel blocks, use one of: clear, cancel, wipe, or drop followed by what you want cleared ("clear my schedule for today", "cancel tomorrow's calendar"). The scheduler will only remove its own non-locked entries; manual ops entries are left in place.
  • Working hours bound everything. Agents will not schedule outside the start/end times in My Hours for the day in question. Update those first if your availability changes.
  • Context is your private state. Mention constraints inline ("I'm out running errands until 5pm"); suggesters and the scheduler read the recent chat history when deliberating.
  • Stuck "thinking"? The Stop agents button beside the chat marks every deliberating agent as committed — useful if a cycle is taking too long or the API is throttled.
  • Report a confusing reply with the small Report link beside any chat message. Reports are saved to disk as cleartext JSON for later diagnosis — no content is forwarded anywhere.

What each agent does in a cycle

  1. Reset — the scheduler deletes its own non-locked future entries for the target date.
  2. Suggest — every relevant suggester posts a SUGGESTION…END block to chat in priority order.
  3. Schedule — the scheduler reads each suggester's latest block and commits a final PLAN…END, creating tickets and calendar entries.

Ambiguous prompts (no clear date, no broadcast, no removal verb) fall back to an LLM relevance check per suggester. The scheduler always engages when a cycle runs, so your calendar will always reflect the latest committed plan.
